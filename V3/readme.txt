此版本开始解析请求内容
目前的测试,浏览器输入地址请求服务端后,服务端读取的请求内容只有请求行和消息头两部分.因此
在ClientHandler中我们先将这两部分的内容分别读取出来,并逐一解析保存.
实现:
1:在ClientHandler第一步解析请求中
  1.1:创建三个变量:
        String method
        String uri
        String protocol
      用来保存请求行中的三部分内容.
  1.2:利用之前实现的读取一行字符串的操作,先读取客户端发送过来的第一行内容,这一行应当
      就是请求行.将其按照空格拆分为三部分,并分别复制给上述三个变量完成请求行的解析工作
